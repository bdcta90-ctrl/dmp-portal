# Product Requirements Document - AI 자동차 손해사정 의사결정 지원 포털

**Author:** Guseo
**Date:** 2026-03-18

## Executive Summary

자동차 보험 손해사정 시장의 손해율은 정부 권고 적정 수준(80%)을 상시 초과하여 87~90%에 이르고 있으며, 85% 이상은 기업의 직접적 손실로 이어진다. 부품가·공임비·첨단기술 비용은 구조적으로 증가하는 반면, 보험료 인상은 고객 이탈 리스크로 제한된다. 이 상황에서 손해율을 낮출 수 있는 유일한 수단은 **손해사정 의사결정의 품질을 체계적으로 끌어올리는 것**이다.

현재 손해사정사의 업무 처리는 개인의 경험치·역량·성향에 전적으로 의존한다. 더 효과적인 처리 방법이 존재하지만 모르거나, 알아도 실행하지 않거나, 리스크를 예측하지 못하는 상황이 빈번하다. 지식 공유는 단절되어 있고(선배도 바쁘고, 실적 경쟁으로 공유 기피), 이로 인해 동일 사건이 담당자에 따라 전혀 다른 결과를 낳는다.

**AI 자동차 손해사정 의사결정 지원 포털**은 이 문제를 해결한다. 배당건 접수부터 처리 완료까지 4단계(견적 산정→대인 피해→과실 산정→처리 방법 제안) 전 과정에서 AI가 **데이터 근거 기반 의사결정을 실시간으로 지원**한다. 손해사정사(실무자)는 복잡한 사건도 즉시 최적 처리 전략을 받고, 팀장/센터장(관리자)은 팀원별 강점·약점 파악, 실적 관리, 민원 사전 대응이 가능해진다.

타겟 고객은 **DB자동차손해사정**이며, kt ds의 전략적 결정으로 추진한다.

### What Makes This Special

기존 보험 AI 시도들은 서류 검토·OCR·자동 분류 등 **보조 업무 자동화** 수준에 머물러 있다. 이 제품은 **손해사정사가 실제 업무를 처리하는 의사결정 과정에 직접 개입하는 시장 최초의 시도**이다.

핵심 기술적 차별화는 **AI Ready Data** 구축이다. 약관·판례·경험에 흩어진 손해사정 암묵지를 온톨로지로 형식지화하여(49개 추론 규칙, 401개 KNIA 사고유형, 801개 연쇄 손상 체인, 12종 수리방법, 12단계 차종 배율), LLM이 실시간으로 의사결정을 지원할 수 있는 구조를 만든다. 단순 RAG(문서 검색)가 아닌, **온톨로지 추론 → 지식 그래프 → AI 분석**이라는 3단계 파이프라인으로 정확도와 설명 가능성을 동시에 확보한다.

## Success Criteria

### User Success

- 배당건 처리 시간 **50% 단축** (기존 평균 대비)
- 신규 손해사정사도 3일 내 시스템 활용하여 **베테랑 수준 의사결정** 가능
- 견적 검증에서 과다청구 **건당 평균 ₩200만원 이상 절감** 식별
- "이 시스템 없이 업무하기 어렵다" — 사용자 만족도 **4.0/5.0 이상**

### Business Success

- 손해율 **80% 내외 유지** (현재 87~90% → 목표 80%)
- 지표 관리 주기: **월/분기/반기/연** 단위 리포팅
- ROI 측정: **비용 절감액** + **손해율 변화** 기준
- DB자동차손해사정 파일럿 후 **6개월 내 손해율 3%p 이상 감소**

### Technical Success

- Agent별 응답 속도 **15초 이내** (견적/대인/과실/처리 각각)
- AI 추천 vs 실제 최적 처리 **일치율 90% 이상**
- 동시 사용자 **10명** (손해사정사 9명 + 팀장 1명)
- 시스템 가용성 **99.5%** (업무시간 기준)

## Product Scope

### MVP - Minimum Viable Product

**4대 핵심 Agent (반드시 포함):**
1. **견적 산정 Agent** — 부위별 수리/교환 판단, AI 견적 검증, ADAS 캘리브레이션
2. **대인 피해 Agent** — 상해등급별 한도 조회, 위자료 산정, 복수 피해자 개별 처리
3. **과실 산정 Agent** — KNIA 사고유형 매핑, 8요소 과실 보정, 증거 기반 신뢰도
4. **처리 방법 Agent** — 4시나리오(최적/공격/안전/전략), 고객 성향 반영, 비용 시뮬레이션

**필수 데이터:**
- 차량DB (60,329건), 부품DB (48,637건), 서비스센터DB (7,846건), 렌터카DB (2,015건)
- 온톨로지 추론 규칙 49개, KNIA 사고유형 401건, 연쇄 손상 801건

**필수 인프라:**

## User Journeys

### Journey 1: 손해사정사 "이현수" — 복잡한 배당건 처리

**인물:** 이현수 (28세, 경력 3년차 손해사정사, 견적·처리에 강하지만 대인 경험 부족)

**Opening Scene:**
월요일 오전 9시. 이현수는 시스템에서 새 배당건 알림을 받는다. 동시에 프린터에서 접수 내용이 출력된다. "교차로 골목길 충돌, BMW 7시리즈 vs 현대 그랜저, 대인 5명." 수리비 청구가 ₩25,000,000이다. 경력 3년차인 그에게도 부담되는 건이다. 예전 같으면 선배에게 물어봤겠지만, 선배도 바쁘고 실적 경쟁 때문에 자세히 알려주지 않는다.

**Rising Action:**
이현수는 AI 포털에 접속한다.
1. **견적 산정 Agent** — 타사 BMW 수리비 ₩25M을 입력하자, AI가 부위별로 "범퍼 판금 가능→교체 청구 (₩1.7M 과다)", "렌트 7시리즈→5시리즈 정정 (₩3.15M 절감)"을 15초 내에 분석해준다. 근거와 함께.
2. **대인 피해 Agent** — 5명 피해자를 입력하자, 각 등급별 한도·위자료·과실상계가 자동 산출된다. 대인 경험이 부족한 그도 즉시 적정 합의금 범위를 파악한다.
3. **과실 산정 Agent** — 교차로 기본 50:50에서 수정요소(신호, 도로, 증거)를 반영하여 AI가 최종 과실을 제안한다.
4. **처리 방법 Agent** — 4가지 시나리오(최적/공격/안전/전략)가 비용·기간·고객만족도와 함께 카드로 제시된다.

**Climax:**
이현수는 "전략안: 미수선 + 제휴센터 + 대인 일괄 패키지"를 선택한다. 기존이라면 생각도 못했을 방법이다. 총 비용 ₩12.5M — 타사 주장대로 처리하면 ₩35.4M이었을 건을 **₩22.9M 절감**하는 결과를 낸다.

**Resolution:**
이현수의 건당 절감액이 팀 최고를 기록한다. 경력 3년차지만 베테랑 수준의 의사결정을 해낸 것이다. 다음 배당건이 와도 더 이상 불안하지 않다.

### Journey 2: 팀장 "김영수 센터장" — 팀 관리 + 민원 사전 대응

**인물:** 김영수 (45세, 센터장, 9명의 손해사정사를 관리, 매월 손해율 리포트를 경영진에 보고)

**Opening Scene:**
수요일 오전 8시 30분. 김영수 센터장은 출근 후 가장 먼저 AI 포털의 KPI 대시보드를 연다. 오늘의 전체 배당건, 종결률, 민원건, 고액건을 한눈에 확인한다.

**Rising Action:**
1. **전체 현황 확인** — 오늘 배당 162건, 종결률 37.5%, 민원 3건, 고액건(₩10M+) 5건. 어제보다 민원이 1건 늘었다.
2. **민원 사전 대응** — 민원 예상건 RPT-011(횡단보도 보행자 접촉)을 클릭한다. AI가 사기 의심 근거(72시간 지연 병원, 18개월 내 타사 2건)와 함께 "SIU 검토 필요"를 표시한다. 김 센터장은 담당 조소영 사정사에게 사전 대응 지시를 내린다.
3. **팀원 실적 평가** — 팀원별 미수선 처리율, 보험사기 적발 건수, 종결률, 처리건수를 확인한다. 정태우 사정사의 종결률이 낮다 — 대인 처리에서 막혀있는 건이 3건이다. 김 센터장은 대인에 강한 박서연 사정사와 협업을 지시한다.
4. **고액건 모니터링** — ₩25M BMW 건(RPT-003)의 진행 상황을 확인한다. AI 적정 ₩18.7M으로 ₩11.75M 절감 예상. 협의 중 상태. 기한 내 종결될지 체크한다.

**Climax:**
월말 리포트 시점. 김 센터장은 대시보드에서 이번 달 손해율을 확인한다 — **83.2%**. 지난달 86.5%에서 3.3%p 감소. AI 포털 도입 전 평균 87~90%와 비교하면 확실한 개선이다.

**Resolution:**
경영진 보고에서 김 센터장은 데이터 근거와 함께 성과를 보여준다. "AI 포털 도입 후 건당 평균 ₩180만원 절감, 종결률 15% 향상, 민원 발생률 40% 감소." 다음 분기 목표인 80% 달성이 현실적으로 보인다.

### Journey 3: 경영진 "DB자동차손해사정 임원" — 경영 대시보드

**인물:** 박상무 (52세, DB자동차손해사정 경영진, 전국 센터의 손해율·수익성 관리)

**Opening Scene:**
분기 경영회의 전날. 박상무는 경영 대시보드에 접속한다. 전국 센터별 손해율 추이, AI 포털 활용률, 절감 실적을 확인해야 한다.

**Rising Action:**
1. **전국 손해율 현황** — 파일럿 센터(김영수 팀)의 손해율이 87% → 83%로 4%p 감소한 것을 확인한다. 비파일럿 센터들은 여전히 87~90%.
2. **ROI 분석** — AI 포털 도입 비용 대비, 파일럿 센터에서 월 ₩2.3억 절감 효과 확인. 연환산 ₩27.6억. 투자 대비 ROI 350%.
3. **확대 의사결정** — 파일럿 결과를 기반으로 전국 확대 일정을 검토한다.

**Climax:**
분기 경영회의에서 박상무는 "AI 포털 파일럿 결과: 손해율 4%p 감소, 건당 ₩180만원 절감, ROI 350%"를 보고한다. 전국 확대가 결의된다.

**Resolution:**
보험료 인상 없이 손해율을 관리할 수 있는 수단을 확보했다. 고객 이탈 리스크 없이 수익성을 회복하는 경로가 열렸다.

## Domain-Specific Requirements

### Compliance & Regulatory (법규·규제)

**자동차보험 관련법:**

## Innovation & Novel Patterns

### Detected Innovation Areas

**1. 의사결정 직접 개입 (시장 최초)**
- 기존 보험 AI: 서류 검토, OCR, 자동 분류 → **보조 업무 자동화**
- 이 제품: 견적→대인→과실→처리 4단계에서 **의사결정 자체를 지원** → 시장에 선례 없음
- 핵심 차별화: AI가 "이렇게 처리하세요"라고 근거와 함께 추천

**2. 온톨로지 기반 AI Ready Data**
- 기존 접근: RAG(약관 PDF 검색) → 문맥 없는 텍스트 반환, 계산 불가
- 이 제품: 49개 추론 규칙 + 401개 사고유형 + 801개 연쇄 손상을 **온톨로지로 구조화**
- 결과: LLM이 단순 검색이 아닌 **추론**을 수행 (예: R02 전손 → R18 잔존물 → R19 할부 연쇄 발동)

**3. 전략안 (대인 미접수 등 암묵지의 형식지화)**
- 기존: 베테랑 손해사정사만 아는 노하우 (예: "100:0으로 양보하되 대인 미접수하면 총비용이 더 낮다")
- 이 제품: 이런 **암묵지를 규칙+시나리오로 형식지화**하여 신입도 활용 가능

### Market Context & Competitive Landscape

|-----------|----------|------|-------------|
| 보험개발원 AOSa | 사진 AI → 견적 자동화 | 견적만, 의사결정 미지원 | 4단계 전 프로세스 커버 |

### Validation Approach

1. **파일럿 검증** — DB자동차손해사정 1개 팀(10명)에서 3개월 운영

## Functional Requirements

### FR-01: 견적 산정 Agent

**입력:**
- 차량 정보: 자사/타사 구분 → 국산/외산 → 제조사 → 모델 → 연식 → 주행거리
- 파손 정보: 8카테고리 70+ 부위 중 선택 + 심각도(경미/중간/심각/전손)
- 사진: 최대 10장 업로드
- UC6(3중추돌): A/B/C 3대 차량 지원

**처리:**
- 부위별 수리/교환 판단 (소재: 금속→판금, 플라스틱→교환, 유리→교환필수)
- 비용 계산: 부품비(PP 테이블 × 심각도 × 차종배율 × 차령감가) + 공임비 + 도장비
- ADAS 캘리브레이션 대상 자동 감지 (8종 부위)
- 미수선 현금정산 산출 (견적의 70~80%)
- 연쇄 손상 자동 추가 (801건 체인 기반)
- 온톨로지 추론 실행 (49규칙)
- LLM 분석 리포트 생성

**출력:**
- 견적 카드 (부품비/공임비/도장비/합계)
- 부위별 세부 테이블
- AI 분석 리포트 (3탭: 타사차량/자사차량/보험사총지출)
- 온톨로지 추론 체인 (발동 규칙 + 근거)
- SMS/일보 모달

**성능:** 15초 이내 응답

**입력:** 전체 배당건 데이터

**표시 항목:**
- 실시간 배당 유입 (일 목표 175건 대비)
- 종결률, 민원건, 고액건
- 팀원별 실적: 미수선 처리율, 보험사기 적발, 종결률, 처리건수
- 절감 실적: 총절감액, 평균절감률, TOP 절감건
- 우선순위별 카드 (긴급/높음/보통/낮음) → 클릭 시 상세
- 사기의심 근거 표시

### FR-06: 데이터 뷰어

- 10개 DB 탭 (차량/서비스센터/렌터카/부품/약관기준/추론규칙/사고유형/수리방법/연쇄손상/차종배율)
- 전체 데이터 자동 로드 (JSON fetch)
- 검색 + 페이지네이션(100행) + xlsx 다운로드
- 행 클릭 → 상세 패널
- 데이터 구조 맵 (7카테고리)
- 온톨로지 센터

## Technical Assumptions & Constraints

### Assumptions
1. DB자동차손해사정이 파일럿 팀(10명)을 제공한다
2. LLM API(Claude) 비용은 kt ds가 부담한다
3. 프로토타입(시연버전)의 데이터·규칙·시나리오를 MVP 기반으로 활용한다
4. 약관 데이터는 2025.08.16 개정 기준이며, 개정 시 수동 업데이트한다
5. 배당건 입력은 MVP에서 수동이며, Growth에서 시스템 연동한다

### Constraints
1. 클라이언트에서 직접 LLM API 호출 금지 (백엔드 프록시 필수)
2. 개인정보(피해자 성명·상해등급)는 LLM에 전송 시 익명화 처리
3. KIDI 차량기준가액은 라이선스 없이 접근 불가 → MVP에서는 수동 입력
4. 동시 사용자 10명 초과 시 아키텍처 재설계 필요
5. 온톨로지 규칙 추가 시 충돌 검증 필수 (overrides/conflictsWith 관계)
